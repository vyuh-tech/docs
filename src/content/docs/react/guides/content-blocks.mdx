## Introduction

Content blocks are the building blocks of your Vyuh React application. They
represent different types of content that can be configured in your CMS and
rendered in your React application. The `@vyuh/react-feature-system` package
provides a comprehensive set of content blocks that you can use out of the box.

For an overview of how content blocks fit into the broader Vyuh framework, see
the [Framework Overview](/react/guides).

## Rendering Content Blocks

All content blocks in Vyuh are rendered using the content plugin's `render`
method:

```tsx
import { useVyuh } from '@vyuh/react-core'

function MyComponent() {
  const { plugins } = useVyuh()

  // Render any content block by passing its object representation
  return plugins.content.render(contentBlockObject)
}
```

## Core Content Blocks

### Route

Routes are page-level content containers that organize content into regions.

```tsx
// Route content object
const routeContent = {
  _type: 'vyuh.route',
  title: 'Home Page',
  regions: [
    {
      identifier: 'body',
      items: [
        // Content blocks in this region
      ],
    },
  ],
}

// Render with content plugin
plugins.content.render(routeContent)
```

### Card

Cards are versatile content blocks for displaying concise information with
optional actions.

```tsx
// Card content object
const cardContent = {
  _type: 'vyuh.card',
  title: 'My Card',
  description: 'This is a card component',
  image: { _ref: 'image-123' },
  action: {
    _type: 'vyuh.action.navigation',
    path: '/details',
  },
}

// Render with content plugin
plugins.content.render(cardContent)
```

### Group

Groups organize multiple content items in different arrangements.

```tsx
// Group content object
const groupContent = {
  _type: 'vyuh.group',
  title: 'Featured Items',
  description: 'A collection of featured items',
  items: [
    // Array of content items
  ],
  layout: {
    _type: 'vyuh.group.layout.carousel',
  },
}

// Render with content plugin
plugins.content.render(groupContent)
```

### Portable Text

Portable Text renders rich text content with embedded blocks. This follows the
[Portable Text](https://github.com/portabletext/portabletext) specification.

```tsx
// Portable Text content object
const portableTextContent = {
  _type: 'vyuh.portableText',
  blocks: [
    {
      _type: 'block',
      style: 'normal',
      children: [{ _type: 'span', text: 'This is rich text content' }],
    },
  ],
}

// Render with content plugin
plugins.content.render(portableTextContent)
```

### Conditional Content

Conditional Content renders different content based on specified conditions.

```tsx
// Conditional Content object
const conditionalContent = {
  _type: 'vyuh.conditional',
  condition: {
    _type: 'vyuh.condition',
    configuration: {
      _type: 'vyuh.condition.boolean',
      value: true,
    },
  },
  cases: [
    {
      value: 'true',
      item: {
        // Content to show when condition is true
        _type: 'vyuh.card',
        title: 'Condition is true',
      },
    },
    {
      value: 'false',
      item: {
        // Content to show when condition is false
        _type: 'vyuh.card',
        title: 'Condition is false',
      },
    },
  ],
  defaultCase: 'false', // Default case if condition evaluation fails
}

// Render with content plugin
plugins.content.render(conditionalContent)
```

### Conditional Route

Conditional Routes render different routes based on conditions.

```tsx
// Conditional Route object
const conditionalRoute = {
  _type: 'vyuh.conditionalRoute',
  condition: {
    _type: 'vyuh.condition',
    configuration: {
      _type: 'vyuh.condition.boolean',
      value: true,
    },
  },
  cases: [
    {
      value: 'true',
      item: {
        // Route to show when condition is true
        _ref: 'route-123',
      },
    },
    {
      value: 'false',
      item: {
        // Route to show when condition is false
        _ref: 'route-456',
      },
    },
  ],
  defaultCase: 'false', // Default case if condition evaluation fails
}

// Render with content plugin
plugins.content.render(conditionalRoute)
```

### API Content

API Content fetches and displays data from external APIs.

```tsx
// API Content object
const apiContent = {
  _type: 'vyuh.apiContent',
  showPending: true,  // Show visual feedback during loading
  showError: true,    // Show visual feedback for errors
  configuration: [
    {
      _type: 'your-api-configuration-type', // Custom API configuration type
      // Configuration properties specific to your API implementation
    }
  ]
}

// Render with content plugin
plugins.content.render(apiContent)
```

<Aside type="note">
  API Content requires a custom API configuration implementation that extends the `APIConfiguration` class. This allows you to define how to fetch data and render the results.
</Aside>

### Divider

Dividers create visual separation between content sections.

```tsx
// Divider content object
const dividerContent = {
  _type: 'vyuh.divider',
  thickness: 1,     // Thickness in pixels
  indent: 16        // Indent from edges in pixels or CSS units
}

// Render with content plugin
plugins.content.render(dividerContent)
```

### Document

Document content blocks represent structured documents with content items.

```tsx
// Document content object
const documentContent = {
  _type: 'vyuh.document',
  title: 'Documentation',
  description: 'A comprehensive guide',
  items: [
    // Array of content items that make up the document
    {
      _type: 'vyuh.portableText',
      blocks: [/* Portable text content */]
    },
    {
      _type: 'vyuh.card',
      title: 'Example Card'
    }
  ]
}

// Render with content plugin
plugins.content.render(documentContent)
```

### Document View

Document View loads and displays a document by reference or query.

```tsx
// Document View content object
const documentViewContent = {
  _type: 'vyuh.document.view',
  title: 'Product Documentation',
  loadStrategy: 'reference',  // 'reference' or 'query'
  reference: { _ref: 'document-123' },  // Used when loadStrategy is 'reference'
  // query: { ... }  // Used when loadStrategy is 'query'
}

// Render with content plugin
plugins.content.render(documentViewContent)
```

### Video Player

Video Player renders video content with playback controls.

```tsx
// Video Player content object
const videoPlayerContent = {
  _type: 'vyuh.videoPlayer',
  title: 'Product Demo',
  linkType: 'url',  // 'url' or 'file'
  url: 'https://example.com/videos/demo.mp4',  // Used when linkType is 'url'
  // file: { _ref: 'file-123' },  // Used when linkType is 'file'
  loop: false,
  autoplay: false,
  muted: false
}

// Render with content plugin
plugins.content.render(videoPlayerContent)
```

## Layouts

Each content block can have different layouts that change how it's presented.
Layouts are specified through the `layout` property or can be inferred from the
content structure. For a comprehensive guide on layouts, see the
[Layouts](/react/guides/layouts) documentation.

```tsx
// Card with a specific layout
const cardWithLayout = {
  _type: 'vyuh.card',
  title: 'My Card',
  description: 'This is a card component',
  layout: {
    _type: 'vyuh.card.layout.button',
    // Layout-specific properties
  },
}

// Render with content plugin
plugins.content.render(cardWithLayout)
```

### Available Layouts

Different content types support different layouts:

#### Card Layouts

- `vyuh.card.layout.default`: Standard card with image, title, and description
- `vyuh.card.layout.button`: Card styled as a button for actions
- `vyuh.card.layout.listItem`: Compact layout for use in lists

#### Group Layouts

- `vyuh.group.layout.carousel`: Horizontal scrolling items with navigation
  controls
- `vyuh.group.layout.grid`: Responsive grid layout with configurable columns
- `vyuh.group.layout.list`: Vertical list of items

#### Route Layouts

- `vyuh.route.layout.default`: Standard page layout
- `vyuh.route.layout.singleItem`: Focused layout for a single content item
- `vyuh.route.layout.tabs`: Sectioned content with tab navigation

<Aside type="tip">
  The same content can be displayed differently by simply changing its layout,
  without modifying the content itself.
</Aside>

## Actions

Actions add interactivity to content blocks. They define what happens when a
user interacts with content. For a comprehensive guide on actions, see the
[Actions](/react/guides/actions) documentation.

```tsx
// Navigation action
const navigateAction = {
  _type: 'vyuh.action.navigation',
  title: 'Go to Details',
  linkType: 'url', // Options: 'url' or 'route'
  url: '/details', // Used when linkType is 'url'
  // route: { ref: 'route-123' },  // Used when linkType is 'route'
  navigationType: 'push', // Options: 'push', 'replace', 'go'
}

// Navigation action using route reference
const routeNavigateAction = {
  _type: 'vyuh.action.navigation',
  title: 'Go to Product Page',
  linkType: 'route',
  route: { ref: 'route-123' }, // ObjectReference to a route ID
  navigationType: 'push',
}

// Open URL action
const openUrlAction = {
  _type: 'vyuh.action.openUrl',
  title: 'Visit Website',
  url: 'https://example.com',
  mode: 'platformDefault', // Options: 'platformDefault', 'inApp', 'externalApp'
}

// JavaScript action
const javascriptAction = {
  _type: 'vyuh.action.javascript',
  script: 'console.log("Action triggered")',
  isAsync: false,
}

// Card with an action
const cardWithAction = {
  _type: 'vyuh.card',
  title: 'My Card',
  action: navigateAction,
}
```

## Conditions

Conditions enable dynamic content selection based on various criteria. For a
detailed explanation of conditions, see the
[Conditions](/react/guides/conditions) documentation.

```tsx
// Boolean condition
const booleanCondition = {
  _type: 'vyuh.condition.boolean',
  value: true,
}

// Authentication condition
const authCondition = {
  _type: 'vyuh.condition.authenticated',
}

// Feature flag condition
const featureFlagCondition = {
  _type: 'vyuh.condition.featureFlag',
  flag: 'new-feature',
}
```

## Creating Custom Content Blocks

You can extend the system with your own custom content blocks by creating:

1. A content type interface
2. Layout configurations for the content
3. A content builder to connect the type with its layouts

See the [Custom Content Types](/react/guides/custom-content-type) guide for more
information.
